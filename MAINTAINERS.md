# Maintainers

This document lists the maintainers, contributors, and governance structure for the Zoterho Template project.

## Project Governance

This project follows the **TPCF Perimeter 3: Community Sandbox** governance model:

- **Open Contribution**: Anyone can submit patches
- **Maintainer Review**: Core maintainers review and merge
- **Consensus-Driven**: Major decisions require maintainer consensus
- **Transparent**: All discussions happen in public (GitHub issues/discussions)
- **Meritocratic**: Contributors gain responsibilities through sustained contribution

## Core Maintainers

Core maintainers have commit access and are responsible for:
- Reviewing and merging pull requests
- Triaging issues
- Setting project direction
- Enforcing Code of Conduct
- Security response

## Becoming a Maintainer

### Contributor Path

1. **Contributor**: First merged PR
2. **Regular Contributor**: 3+ merged PRs over 3+ months
3. **Core Contributor**: 10+ merged PRs, sustained engagement
4. **Maintainer**: Nominated by existing maintainer, consensus approval

## Decision-Making Process

### Minor Decisions
- Code reviews, minor bug fixes, documentation updates
- **Process**: Any single maintainer can approve and merge

### Major Decisions
- Breaking changes, new features, architecture changes
- **Process**: Requires consensus (all active maintainers agree or abstain)

### Governance Changes
- Changes to this document, Code of Conduct, contribution process
- **Process**: Requires 2/3 majority of active maintainers + 7-day public comment period

### Conflict Resolution
1. Discussion among maintainers
2. If no consensus, public RFC (Request for Comments)
3. Community input period (14 days)
4. Final decision by majority vote

## Stepping Down

Maintainers can step down at any time:

1. Notify other maintainers (private or public)
2. Transfer active responsibilities
3. Update MAINTAINERS.md
4. Move to Emeritus section (with permission)

No explanation required. We respect life changes and thank you for your service!

## Inactive Maintainers

Maintainers inactive for 12+ months with no communication:

- Moved to Emeritus after attempting contact
- Can return by expressing interest
- Commit access removed for security
